# ocr2md-mcp Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a lightweight MCP server that sends local files to TextIn OCR and returns structured OCR output plus Markdown, including bounded-concurrency batch processing.

**Architecture:** The server is a thin MCP wrapper over a reusable TextIn HTTP client and a small set of pure utility modules. OCR orchestration is separated from transport setup so the batch logic, file resolution, and Markdown rendering can be tested without needing a live MCP connection.

**Tech Stack:** Node.js, TypeScript, `@modelcontextprotocol/server`, `zod`, `fast-glob`, `dotenv`, `vitest`

export async function resolveInputFiles(input: ResolveFilesInput): Promise<string[]> {
  const explicit = input.filePaths ?? [];
  const discovered = input.directory
    ? await fg(input.pattern ?? DEFAULT_FILE_PATTERN, { cwd: input.directory, absolute: true, onlyFiles: true })
    : [];
  return Array.from(new Set([...explicit, ...discovered])).sort();
}

export function buildMarkdownFromResults(results: MarkdownInputResult[], options: MarkdownOptions = {}): string {
  const heading = '#'.repeat(options.headingLevel ?? 1);
  return results.map((result) => `${heading} ${result.filePath}\n\n${result.text || '_No text recognized._'}`).join('\n\n');
}
